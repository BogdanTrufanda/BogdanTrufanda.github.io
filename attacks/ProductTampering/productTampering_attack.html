Product Tampering - Attack

The first step to take in order to complete this type of attack is to login with an existing user and search for a product, o-saft for example, while monitoring with Burpsuite.

As we can see above, we have the GET request that is specific to item searching and we will send it to Repeater in order to make it useful.

Below we can see the GET request in the repeater with a little modification on the 1st line : GET /rest/products/search?q=saft HTTP/1.1. This will allow us to see the information of the product we want to modify and more exactly, the id of that product.

In the next image we will see a GET response in BurpSuite that was send after we have clicked on product on the application in order too see it's reviews. We need this request to modify so we will send it in Repeater.

You can now see a whole transformation of the request we have logged because he will transform into an API request in order to be able to set the needed information on the targeted item. We will just take the right side informations and put it at the end of the request from the left.

After modifying the GET request from the Repeater, we will see that it will change after the first send and now we will have the information from the right on the left also.

Once the send was successfull we can delete all the information excepting the description because it contains the href we need to modify. On top to that just add the Content- Type from the right and there you go, you have changed the hyper link of that product.
