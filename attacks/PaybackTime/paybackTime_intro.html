Payback Time - Intro

This attack is an Improper Input Validation as Admin Registration but for this one we have to trick somehow the application to owe us some money in the shopping cart..

What is Improper Input Validation?

When software does not validate input properly, an attacker is able to craft the input in a form that is not expected by the rest of the application. This will lead to parts of the system receiving unintended input, which may result in altered control flow, arbitrary control of a resource, or arbitrary code execution.

In order to successfully perform such an attack we have two main checkpoints to achieve:

  • Login and order something from the shop on more than 1 quantity.
  • Think of the simplest possible implementations to reuse the request for the product we order to perform an unusual request that will make the shop owe us some money.
