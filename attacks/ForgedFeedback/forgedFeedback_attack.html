Forged Feedback - Attack

The first step to take in order to complete this type of attack is to login with an existing user and post a feedback while monitoring with Burpsuite.

As we can see above, we have a POST request with our feedback and other specific fields like userId, captchaId and captchaAnswer.

In this case we can assume that we could change the fields within POST request with another user's email and things are done.

In this case we will get the POST request and send it to Repeater in BurpSuite in order to modify the values of needed fields. As we can see in demo, the user's email is hidden and all we can see is the last letter and the domain of the email adress.In this case we can set an anonymus feedback and so we don't have to match any userID with any userEmail.

You can see that the POST request is successfull meaning that we have posted an anonymus feedback from our test account.

If we are checking the score board, we will have the challenge successfull.
