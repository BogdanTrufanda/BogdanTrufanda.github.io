Repetitive Registration - Vulnerability

There are few steps to follow in order to test if the application has or not this vulnerability.

1. Go to user registration and fill everything up except for Password and Repeat Password.

2. Type e.g. 12345 into the Password field.

3. Now type 12345 into the Repeat Password field. While typing the numbers you will see a Passwords do not match error until you reach 12345.

4. Finally go back to the Password field and change it into any other password. The Repeat Password field does not show the expected error.

5. Submitting the form will create your user and the valid password will be the one from Password field.
