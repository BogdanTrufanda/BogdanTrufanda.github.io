Enforce a redirect to a page
you are not supposed to redirect to



This challenge is about redirecting to an entirely disallowed different location.

  • This challenge is about redirecting to an entirely disallowed different location.
  • The application will only allow you to redirect to whitelisted URLs.
  • Tampering with the redirect mechanism might give you some valuable information about how it works under to hood

Steps:

  • Pick one of the redirect links in the application, e.g. http://aset2020.herokuapp.com/redirect?to=https://github.com/bkimminich/juice-shop from the GitHub-button in the navigation bar.
  • Trying to redirect to some unrecognized URL fails due to allowlist validation with 406 Error: Unrecognized target URL for redirect.
  • Removing the to parameter (http://aset2020.herokuapp.com/redirect) will instead yield a 500 TypeError: Cannot read property 'indexOf' of undefined where the indexOf indicates a severe flaw in the way the allowlist works.
  • Craft a redirect URL so that the target-URL in to comes with an own parameter containing a URL from the allowlist, e.g. http://aset2020.herokuapp.com/redirect?to=http://kimminich.de?pwned=https://github.com/bkimminich/juice-shop
