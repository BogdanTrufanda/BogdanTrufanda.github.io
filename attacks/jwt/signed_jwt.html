Forge an almost properly RSA-signed JWT token

Forge an almost properly RSA-signed JWT token that impersonates the (non-existing) user rsa_lord@juice-sh.op. Getting your hands on the public RSA key the application employs for its JWTs is mandatory for this challenge.

  • Use a brute-forcing directory browsing tool to identify http://localhost:3000/encryptionkeys as having directory listing enabled.
  • Download the application's public JWT key from http://localhost:3000/encryptionkeys/jwt.pub.
  • In the BApp Store tab under the Extender tab within Burp Suite find and install the JSON Web Token Attacker extension (aka JOSEPH)b
  • Send any captured request that has an Authorization: Bearer token to Burp's Repeater.
  • Once in Repeater, click the JWS tab, then the Payload tab beneath and modify the email parameter to be rsa_lord@juice-sh.op
  • Next, click the Attacker tab, select Key Confusion, then click Load.
  • Paste in the contents of the jwt.pub file without the -----BEGIN RSA PUBLIC KEY----- and -----END RSA PUBLIC KEY----- lines.
  • Upload the new changes, and then send the forged request.
